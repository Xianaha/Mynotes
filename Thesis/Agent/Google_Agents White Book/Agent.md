# Agent

## 什么是Agent？
从最根本的意义上讲，生成式AI agent(智能体)可以被定义为一种应用程序，它通过观察世界并利用其可支配的工具来采取行动，以实现某个目标。

### Agent的组成部分
Agent由三个主要组成部分组成：
- 模型（Models）
- 工具（Tools）
- 编排层（Orchestration Layer）

### 模型
作为Agent流程的集中决策者使用的语言模型。有以下几个特点：
- Agent使用的模型可以是一个，也可以是多个；其大小（指参数）可以是任意规模。
- 能够遵循基于指令的推理和逻辑框架——如ReAct、思维链（Chain-of-thought）或思维树（Tree-of-Thoughts）。
- 模型可以是通用大模型，也可以是专用小模型，也可以是多模态的模型。也可以是针对特定agent架构的需求进行微调的。  

为了获得更好的生产效果，应用到的模型，理想情况下，是针对计划在认知架构中使用的工具相关的数据及标签进行训练的。不过通常模型不是使用agent的特定配置（即工具选择、编排、推理设置）进行训练的。
### 工具
工具弥补了模型无法与外部世界交互的缺陷，使Agent能够与外部数据和服务进行交互，同时解锁了超出基础模型自身能力的更广泛的操作范围。工具通常与常见的Web API方法（如GET、POST、PATCH、DELETE）保持一致。  
例如，工具可以更新数据库中的客户信息或获取天气数据，以影响Agents向用户提供的旅行建议。借助工具，agents可以访问和处理现实世界的能力，这使他们能够支持跟专业的系统，如检索增强生成（RAG）。        
### 编排层
编排层描述了一个循环过程，该过程控制着agent如何获取信息、进行内部推理，并利用该推理来指导下一步行动或决策。通常，该循环会持续进行，直到agent达到其目标或到达某个停止点。  
编排层的复杂度取决于其执行的任务而异。
### Agent与LLM的区别
|模型（Model）|Agent|
|:-------|:--------|
|知识仅限于他们的训练数据中可获取的内容|知识通过工具与外部系统的连接得以扩展|
|基于用户查询进行单一推理\预测，除非模型中明确实现，否则不会进行会话管理、历史记录或链接上下文。（即聊天记录）|管理会话历史（即聊天记录），以便根据用户查询和在编排层做出的决策进行多轮推理\预测，在此上下文中，“一轮”被定义为交互系统与agent之间的交互。（即1个传入事件\查询和1个agents响应）|
|无原生工具实现。|工具在agent架构中是原生实现的。|
|未实现原生逻辑层。用户可以以简单问题形式构建提示，或使用推理框架（如CoT、ReAct等）构建复杂提示，以引导模型进行预测。|采用如CoT，ReAct等推理框架，或如LangChain等与构建Agents框架的原生认知架构。|

## Agent的具体运作
对于Agent来说，完成每一个任务都需要设计一些列的规划。执行和调整过程——包括收集信息、内部推理、执行等等。  
在流程的每个阶段，agent都会根据需要进行调整，随着信息的查询或用户的输入和反馈，不断优化决策，并利用历史结果来确定下一步的决策。
agent认知架构的核心是编排层，负责维护记忆、状态、推理和规划。它利用快速发展的提示词工程及相关框架来指导推理和规划，使Agent能够更有效地与环境进行交互。以下有几个最受欢迎的几个框架和推理技术：（本文发表前）
- ReAct
ReAct是一个提示工程框架，它为语言模型提供一种思维过程策略，使其能够根据用户查询进行推理并采取行动，无论是否提供上下文实例。

- 思维链（CoT）
一种提示工程框架，通过中间步骤实现推理能力。CoT有各种子技术，包括自一致性、朱东提示和多模态CoT，每种技术都有其优缺点，具体取决于应用。

- 思维树（ToT）
一个非常适合探索或战略前瞻任务的提示工程框架。它概括了思维链提示，并允许模型探索各种思维链，这些思维链可以作为使用语言模型解决一般问题的中间步骤。

以上推理技术，Agent可以使用其中一种或多种，以此为给定的用户请求选择下一个最佳的行动。

虽然模型可以根据其先验知识猜测答案（即产生幻觉），但是它还是选择使用工具（Flight）来搜索实时外部信息。这些额外的信息被提供给模型，使其能够基于真实数据做出更明智的决策，并将这些信息总结反馈给用户。
